# 模块

~~~markdown
Python的执行架构：
1. 模块 + 顶层文件
		顶层文件只有一个：用于调用其他模块
		模块可以有多个：用于模块之间相互调用
2. 模块或顶层文件，本质都是文件 
		模块：.py  .pyd  .pyo  .pyc .pyw  .so
~~~

* 模块的概念

~~~markdown
1. 可以将代码量较大的程序，分割成多个有组织，彼此独立但又能相互调用的代码片段，这些自我包含的有组织的代码就是模块
		文件
2. 包：
		1. 包是一个目录
		2. 包的创建
			1. 创建一个文件夹
			2. 在该文件夹内，创建__init__.py 的文件
3. 包和模块的关系：
		包>模块
		包是模块不可分割的一部分（包相当于模块的姓  模块名相当于模块的名字）
~~~

* import 语句

~~~markdown
1.  import  包名.子包名.模块名 
		1. 必须将所有的名字都写全（模块的全名）
		2. 如果要使用模块内部的属性，需要写全名，如果包的层级过多，则使用比较麻烦
		3. 相对比较安全，不容易发生命名冲突
		4. 导入级别：模块级别
2. import 包名.子包名.模块名 as 别名
		1. 使用方便
		2. 容易发生命名冲突
		3. 如果起了别名，原名就不存在了
3. from 包名.子包名.模块名 import 属性
		1. 导入级别：属性级别
		2. 容易发生命名冲突
4. from 包名.子包名.模块名 import 属性 as 别名
		1. 用于解决命名冲突问题from 包名.子包名.模块名 import 属性
~~~

* 模块的作用

~~~markdown
1. 提高代码的复用性
		模块可以将繁琐的代码封装到模块中
		一处实现，处处使用
2. 提高代码的可读性
		将繁琐的封装通过模块的形式整体调用，结构简单易懂
3. 减少代码冗余


补充：
1. 模块如果被其他模块导入，在执行的时候，会自动加载一次，加载会触发一次执行
~~~

* __name\_\_

~~~markdown
1. __name__是当前模块默认的属性
		不需要手工定义
2. 当__name__是在当前模块使用时：
		此时程序的流程就在当前模块
		此时__name__返回的是：字符串 __main__
3. 当__name__是在别的模块流程使用时
		此时__name__返回的是对应模块的模块名
4. 作用：
		开发测试，在当前模块下，测试自己的代码，不影响别人
~~~

~~~python
# 目标：只有自己做测试的时候才打印，别人调用我自己，不打印，不给别人添麻烦
if __name__=='__main__':
    for i in range(10):
        print('hello world')
~~~

* 模块的分类

~~~markdown
1. 内置模块：
		Python自带的官方的模块，体现为直接可以import 模块名 
2. 自定义模块：
		自己写的模块，往往需要加上包： import 包名.子包名.模块名
3. 第三方模块：
		一般从Pypi社区下载的模块
		Python的模块安装： 
            pip install 模块名
            pip: Python自带的模块安装工具
		
		Pypi社区：www.pypi.org  ---世界级网站（所有的第三方模块）
		所有的第三方模块，默认下载到了 site-packages 文件夹下
~~~
